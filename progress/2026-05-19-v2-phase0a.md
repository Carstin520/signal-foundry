## Architecture Decisions

- Hyperliquid's public WebSocket `WsBook` message exposes `coin`, `levels`, and `time`, but not a conventional incrementing book sequence. For Phase 0a, `seq` is the venue-provided millisecond `time` field, and gap detection measures deltas in that venue clock. This keeps the schema compatible with Section 2.2 of `goal/v2-execution-plan.md` without inventing an internal sequence.
- Reconnect resume is best-effort. The capture resubscribes after disconnect, keeps the last observed `seq` per stream and market, and skips strictly older events. Hyperliquid does not expose a replay cursor for public `l2Book`; missing intervals are therefore surfaced by the gap checker instead of hidden.
- Parquet writes use DuckDB's local Parquet writer rather than adding PyArrow or a new storage service. This follows the storage constraint in `goal/v2-execution-plan.md`: DuckDB plus partitioned Parquet, no new infrastructure.
- The capture stores normalized raw event rows only. It does not commit raw WebSocket payload blobs, secrets, API keys, private keys, DMs, paywalled data, or trading credentials.
